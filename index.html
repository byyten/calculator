The Internal Debate

(the infernal debate)

the brief for this project suggested building a simple calculator that evaluated a pair of numbers and display a result before continuing to evaluate any further inputs which seemed in my mind to be both correct and mathematically incorrect because if the whole expression were to be evaluated using operator precedence the result (from the suggested solution) would often be mathematically incorrect - not a good scenario for a calculator and the planet's/universe's only precise and deterministic language (maybe programming languages could be in there as well with some caveats)

this created a disonance in my mind and thus led me to abandon the brief in that respect and evaluate the input expressions using operator precedence in a more correct interpretation of the math (the calculator will tell you if the answer it produces differs from what javascript 'eval()' produces) by highlighting the result in red however it is still lacking some error checking or disabling of keys if something should be prohibited. In other words it relies on an operator inputing sensible and valid expressions (just as well this isn't ever going to be a product ;)
